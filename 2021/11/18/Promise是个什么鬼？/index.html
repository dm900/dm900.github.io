一、怎么定义？

简单说：异步编程的一种解决方案

二、解决问题

目标：解决网络请求回调地狱

1.什么是回调地狱

异步操作嵌套异步操作，一层套一层，最后就会造成回调地狱函数的产生。

举例：定时器异步请求

  setTimeout(() => {

      setTimeout(() => {
        console("hello two");

          console("hello two");
          console("hello two");
          console("hello two");
        }, 1000);
      }, 1000);
    }, 1000);
  }, 1000);
});

想要做此处理必须添加 resolve 函数调用。而一旦调用 resolve，将要执行 then()函数。可以每次调用作为一次网络请求来处理，这样上面的函数即可以改编成如下，即：

三、什么情况下会用到 Promise

一般情况下，异步请求操作的时候，将会使用 Promise 来对这个异步操作进行封装

在 new Promise 的时候将会执行构造函数内部其实：

  • 1.保存一些状态信息；

  • 2.执行传入的函数；

  • 3.执行传入的函数时，会传入两个参数：“resolve”和“reject”，这两个参数本身又是函数）

  • 4.成功的时候调用 resolve 函数，失败的时候调用 reject 函数。

将请求的代码数据与要处理的代码做一个分离处理

四、Promise 的三种状态

async(异步)操作开始，被包裹到 Promise 中，会有以下三种状态。

1.pending：等待状态，比如正在请求进行网络请求，或者定时器还没到时间；

2.fulfill：满足状态，当我们主动调用 resolve 函数时，就处于该状态，并且会调用.then()函数；

3.reject：拒绝状态，到我们调用 reject 函数时，就处于该状态，并且会回调.catch()函数。

五、Promis 的另一种写法

new Promise((resolve, reect) => {

七、一旦捕捉到错误，其他的不在执行

*以下代码在执行到return Promise.reject(‘err message’)*时,将会到最后的.catch 函数处执行**：
.catch(err =>{
console.log(err)
})

  //第一次发起网络请求
  setTimeout(() => {
    resolve("000");
  }, 1000);
})
  .then(() => {
    console("第1次处理了100行代码逻辑");

  .catch((err) => {

八、自己实现最简单的 Promise

开发中会有这样的问题存在，需要有两个请求回来的数据才能进行下一步，那么如何判断两个请求成功？

第一方案：分别在两个请求开始前自定义布尔变量 isResult 为 false，当每一个请求成功时改变布尔值为 true，定义处理结果函数，先判断两个布尔值是否为真，同时为真进行下一步。

第二方案：Promise 解决，利用 Promise 迭代器

Promise.all([
  new Promise((resolve, reject) => {
    $ajax({
      url: url1,
      success: function () {
        resolve(data);
      },
    });
  }),
  new Promise((resolve, reject) => {
    $ajax({
      url: url1,
      success: function () {
        resolve(data);
      },
    });
  }),
]).then((results) => {
  results[0], results[1];
});
